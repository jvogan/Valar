# Code of Conduct

## Our Standard

Valar is an open-source project for local speech tooling on macOS and Apple Silicon. We expect contributors and maintainers to keep discussion technical, respectful, and constructive.

Examples of behavior that contribute to a positive environment:

- focusing on reproducible technical reports and clear proposals
- being direct without being hostile
- accepting good-faith review, correction, and disagreement
- keeping beginner-facing documentation accurate and non-dismissive

Examples of unacceptable behavior:

- harassment, personal attacks, or discriminatory language
- intentionally hostile review or support interactions
- publishing private exploit details in public before maintainers can triage them
- posting private workstation paths, secrets, or personal data in issues or pull requests

## Scope

This code of conduct applies to:

- the public `Valar` repository
- issues, pull requests, and discussions related to the repo
- project-maintained communication channels for Valar

## Reporting

For routine behavior issues, open a GitHub issue or GitHub Discussion if the matter is not sensitive.

For private conduct concerns, use the private GitHub Security Advisory intake for this repository and note that the report is conduct-related. This is the current private maintainer contact path until a dedicated conduct address is published.

For security issues, do not use public issues. Follow [SECURITY.md](./SECURITY.md) instead.

## Enforcement

Maintainers may remove comments, close threads, or limit participation when behavior makes the project harder to use, maintain, or safely discuss.
